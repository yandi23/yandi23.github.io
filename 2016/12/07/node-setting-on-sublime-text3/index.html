在sublime下配置node环境/代码提示包

发表于 2016-12-07 | 分类于 node

个人比较喜欢用sublime做开发.最近到了node,所以,当然也得在sublime上配置下:

准备

  • 用到插件/软件:node和SublimeText-Nodejs
  • SublimeText-Nodejsgithub地址
  • node下载

node的安装

  • node下载
  • 下载下来后一路点击下一步知道安装结束就可以了
  • 如果修改了安装路径的话还需要配置环境,所以懒人就不要修改安装路径啦
  • 完成后,如果在cmd里面输入node -v返回版本号证明安装成功(将node集成到cmd里面的情况)

从PackageControl中下载node插件

这个是在sublime上面按装插件的第一反应,但是在作者提供的安装方法里面并没有在pc里面安装的步骤,从网上查资料得知,从pc里面安装的插件似乎没有配置文件,也就是不能修改配置.所以这点作罢(但我并没有做验证,懒也),如果你觉得此法可行,可以参照这里创建配置文件

从作者仓库下载压缩包安装

  • SublimeText-Nodejs作者仓库地址

安装步骤

  1. 我们把下载下来的压缩包解压,然后把这个解压后得到的文件夹放进sublime安装目录的Packages文件夹下(目录进入方式:preferences–>Browse Packages)
  2. 在nodejs文件夹内找到Nodejs.sublime-build文件:
    • 将代码 “encoding”: “cp1252” 改为 “encoding”: “utf8”
    • 将代码 “cmd”: [“taskkill /F /IM node.exe & node”, “$file”] 改为 “cmd”: [“node”, “$file”]
      //这里的意思在执行新node程序之前先干掉之前的node进程.看别人的教程这里需要改动,但是我不建议改动,因为个人觉得这样挺好用的
    • 保存文件
  3. 打开文件“Nodejs.sublime-settings”:
    • 将代码 “node_command”: false改为 “node_command”: “D:\\Program Files\\nodejs\node.exe”
      //其实这里就是你的node安装目录下的node.exe,这里必须是双斜杠,因为需要转义
    • 将代码 “npm_command”: false 改为 “npm_command”: “D:\\Program Files\\nodejs\\npm.cmd”
      //这里就是你的npm.cmd的路径,一般都在node安装目录下
    • 保存文件
  4. 测试配置是否成功:
    • 新建一个js文件
    • 写点东西:console.log(“test”),保存
    • ctrl+b执行代码,观察sublime控制台是否有test输出,如果有输入则配置成功

控制台无输出

  • 之前我遇到一次,查遍网络,也没解决,最终发现:是因为文件名是node.js的原因,改了就好了
