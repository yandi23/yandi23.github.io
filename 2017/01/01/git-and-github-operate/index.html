分支冲突

  • 当不同分支的相同文件内容发生不同改变的时候,会发生冲突
发生情况
  • master有a文件
  • dev分支对a文件进行了+”dev”操作,并提交到了版本库
  • master对a文件进行了+”master”操作,并提交到了版本库
  • 这时候合并分支dev,git不知道用谁合并谁,就会产生冲突
处理冲突
  • 需要打开a文件手动处理冲突

git远程操作

创建忽略文件

  • 只能在命令行里面或者webstrom里面创建.开头的文件,不能在window中通过重命名创建
  • 给忽略文件添加忽略内容,把需要忽略的文件夹或者文件名放进.gitignore

创建远程仓库

  • 在github上创建仓库
  • 一般新建的仓库不要有任何东西,不然会发生冲突

本地与远程冲突

  • 需要先把远程pull下来,手动处理冲突,合并之后再推送
  • 或者强制推送,本地覆盖远程不建议使用

在github挂载静态页

  • 挂载静态页必须将代码提交到固定的分支上(gh-pages)
  • 将静态页推送到gh-pages分支上
  • 在github的分支的设置上打开网址即可,网址在仓库setting里面,如果静态页的名称不是index,需要手动输入文件名访问

github操作

  • fork:相当于复制
  • follow:相当于关注这个人
  • star:关注这个项目
  • issue:提问
  • watch:订阅

其他bash命令

  • cd:改变目录
  • ls:打印当前目录子目录
  • ls -al:打印当前目录下的所有文件,包括隐藏文件
  • pwd:打印工作路径
  • mkdir:创建一个文件夹
  • touch:创建一个文件
  • echo:在文件中写入内容
  • cat:查看文件内容
  • vim:用vim打开文件
    • i:inset,在当前位置前插入内容
    • a:在当前位置后插入
    • esc+:q!不保存退出
    • esc+:wq保存退出
  • rm -rf :删除文件夹及子文件/文件夹
  • history:查看所有输入的操作命令
