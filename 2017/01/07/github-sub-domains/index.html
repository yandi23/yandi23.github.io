关于Github Pages绑定二级域名

发表于 2017-01-07 | 分类于 教程

今天捯饬了一天都在为了实现自己的一个需求:

我有一个域名a.com,我有个username.github.io仓库放了一个博客,绑定的是a.com这个域名.现在想让做一个主页,绑定a.com,通过主页可以跳转到博客、demo、简历等,而这三个页面的域名分别为:blog.a.com、demo.a.com、resume.a.com,也就是我想把博客的域名改成blog.a.com,用a.com去绑定主页.<需求完>.

我最初的想法是把东西放一个仓库,目录如下:

查了查,这样无法解析域名,于是换了个思路:

  • 新建一个仓库为homepage,把主页放到这个仓库的gh-pages分支,绑定域名a.com
  • 把博客仓库的域名换成blog.a.com

结果给博客域名换成blog.a.com之后,更换主页域名的时候出现了问题:

于是我把主页绑定的域名删了,发现主页的默认域名是blog.a.com/homepage,似乎是在username.github.io域名的基础上加上仓库名.似乎不能使用二级域名.

在折腾了很久之后我发现:github上所有gh-pages分支上的域名都是在username.github.io这个仓库的域名的基础上加上gh-pages所在仓库的名字.所以,我想到了这样的解决办法:

  • 把主页放在username.github.io这个仓库,绑定a.com域名
  • 把博客放在新建的blog仓库的gh-pages分支,不用管域名
  • 这样主页的域名为a.com,博客的地址为a.com/blog,没有使用二级域名,但是也解决了上面的问题.

上面是github的操作,hexo博客配置也要做修改:

  • 修改url和根目录
  • 更改hexo博客上传的分支

改动这两步就可以了.就可以了.

在搞定这个需求的同时发现了有意思的事:

由于需要把主页放在username.github.io这个仓库,我不会删除删除仓库里的文件,只好忍痛把这个仓库删了,几个月的绿点没了.当我把博客的文件原封不动的传到gh-pages分支上的时候发现以前的绿点又都回来了.似乎这些绿点的记录在本地也有存储,或者说有可能github依据本地的某些内容来标记绿点.
