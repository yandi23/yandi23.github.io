Flex 布局入门

发表于 2017-01-13 | 分类于 整理

最近在自学Flex布局相关的知识,把最近学的东西整理一下.作为学习的总结.

flex布局

特点

  • 布局处理
  • 对齐处理
  • 剩余空间处理

基本概念

  • flex容器:采用flex布局的元素叫做flex容器
  • flex项目:flex容器内的所有子元素自动成为容器成员,称为flex项目
  • 主轴:容器默认的主轴是水平方向从左往右的;主轴开始的位置叫做main start,结束位置叫做main end
  • 交叉轴:和主轴垂直,默认为从上往下;交叉轴开始的位置叫做cross start,结束位置叫做cross end
  • 项目排列方向:项目默认按照主轴排列,单个项目占据的主轴空间叫做main size,占据的交叉轴空间叫做cross size

使用flex布局

  • 设置display:flex|inline-flex
    • flex:块状元素使用
    • inline-flex:行内元素使用
  • webkit内核的需要加上-webkit前缀
  • 设为flex布局之后子元素的float、clear、vertical-align属性都会失效

属性

容器属性

作用在容器上的属性:

  • flex-direction
  • flex-wrap
  • flex-flow
  • justify-content
  • align-items
  • align-content

flex-direction

  • 决定主轴方向(即项目排列方向)
  • flex-direction:row|row-reverse|column|column-reverse
    • row(默认值):主轴为水平,从左到右
    • row-reverse:主轴为水平,从右往左
    • column:主轴为垂直,从上到下
    • column-reverse:主轴为垂直,从下到上

flex-wrap

  • 默认情况下,项目都排在一条轴线上,flex-wrap属性定义了如果一条轴线排不下如何换行
  • flex-wrap:nowrap|wrap|wrap-reverse
    • nowrap(默认):不换行,自动调节项目宽度
    • wrap:换行,在第一行下面出现
    • wrap-reverse:换行,在第一行上面出现

flex-flow

  • flex-flow是flex-direction&&flex-wrap属性的复合写法
  • flex-flow:||

justify-content

  • 定义了项目在主轴上的对齐方式
  • justify-content:flex-start|flex-end|center|space-between|space-around
    • flex-start(默认):向主轴起始方向对齐
    • flex-end:向主轴结束方向对齐
    • center:居中
    • space-between:两端对齐,项目之间间隔相等
    • space-around:每个项目两侧间隔相等

align-items

  • 项目在交叉轴上如何对齐
  • align-items:flex-start|flex-end|center|baseline|stretch
    • flex-start:向交叉轴的起始位置对齐
    • flex-end:向交叉轴的结束位置对齐
    • center:居中
    • baseline:向项目的第一行文字的基线(文字的底部)对齐
    • stretch(默认):如果项目未设置高度或者高度为auto,将拉伸占满整个容器的高度

align-center

  • 定义了多根轴线(多行)的对齐方式,如果项目只有一根轴线(只有一行),该属性不起作用
  • align-content:flex-start|flex-end|center|space-between|space-around|stretch
    • flex-start:向交叉轴的起始位置对齐
    • flex-end:向交叉轴的结束位置对齐
    • center:向交叉轴的中点对齐
    • space-between:向交叉轴的两侧对齐
    • space-around:每根轴线两侧的间隔相等
    • stretch(默认):轴线占满整个交叉轴

项目属性

设置在项目上

  • order
  • flex-grow
  • flex-shrink
  • flex-basis
  • flex
  • align-self

order

  • 定义项目排列顺序,值越小,排列越靠前,默认值为0

flex-grow

  • 定义项目的对剩余空间的放大比例(也可以理解为定义了项目对剩余空间的平分比例),默认为0,即:如果存在剩余空间,也不放大
  • 例子

1.项目1的flex-grow值为1,项目2的flex-grow值为0,剩余空间为200px;
则:
项目1会增加(200/1+0)*1=200的空间,项目2增加(200/1+0)*0=0
2.项目1的flex-grow值为1,项目2的flex-grow值为1,剩余空间为200px;
则:
项目1会增加(200/1+1)*1=100的空间,项目2(200/1+1)*1=100的空间

flex-shrink

  • 定义了空间不够时候项目的缩小比例,默认为1
  • 例子

如果一个容器宽400px,包含有两个宽220px的项目
1.如果第一个项目的flex-shrink=0;第二个项目的flex-shrink=1;
则:
实际上第一个项目宽220-((220+220-400)/1)*0=220px;第二个项目宽220-((220+220-400)/1)*1=180px;
2.如果第一个项目的flex-shrink=1;第二个项目的flex-shrink=1;
则:
实际上第一个项目宽220-((220+220-400)/1+1)*1=200px;第二个项目宽220-((220+220-400)/1+1)*1=200px;

flex-basis

  • 用于定义在分配压缩空间之前,项目占据的主轴空间(main size).浏览器根据这个属性来计算是否有剩余空间.默认值为auto,即项目本身的大小
  • 当容器超出项目空间时候,根据flex-grow来分配空间,这时候flex-basis不会参与运算
  • 当项目超出容器空间的时候,依据flex-shrink和flex-basis来分配压缩空间,公式如下:
    设:所有项目的flex-shrink*flex-basis累加值为n,
    每个项目需要压缩的大小为:

w = (项目的flex-basis (flex-shrink)/n) 溢出值

flex-self

  • 该属性允许单个项目有与其他项目不一样的对齐方式,可覆盖align-items属性.默认为auto,表示继承容器的align-items属性,如果没有容器,则等同于stretch
  • flex-self:auto|flex-start|flex-end|center|stretch|baseline
